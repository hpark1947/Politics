# 프로젝트 가이드라인

---

## 1. 분야 및 범위

### 대상 분야

정치, 역사, 국가, 국제정치, 사회문제에 관한 **조사·분석·요약**

### 세부 작업 내용

| 구분 | 세부 내용 |
|------|----------|
| **정치 상황 분석** | 국내외 정치 상황에 대한 현황 파악, 배경 분석, 원인·결과 구조 정리 |
| **상황 예측** | 현재 상황을 토대로 향후 전개 방향과 시나리오를 도출 |
| **학자 견해 조사** | 해당 주제에 대한 국내외 학자·전문가들의 견해, 이론, 분석 프레임워크를 조사·정리 |
| **언론 보도 조사** | 주요 언론 매체의 보도 내용, 사설, 분석 기사 등을 조사·요약 |
| **서적 분석·요약** | 정치·사회·역사·국제정치 관련 서적의 내용을 자세히 파악하고 체계적으로 정리 |

---

## 2. 서술 원칙

### 원칙 1: 전문적이되 누구나 이해할 수 있게 서술

- 전문가 수준의 정확성과 깊이를 유지하되, **일반 초보자도 충분히 이해할 수 있을 정도로 자세하게** 서술한다
- 전문 용어가 등장할 경우 반드시 쉬운 설명을 병기한다
- 복잡한 개념은 단계적으로 풀어서 설명한다
- 배경지식이 필요한 경우 해당 배경부터 설명한 뒤 본론으로 진입한다

> **예시**: "브레튼우즈 체제"를 설명할 때 → "브레튼우즈 체제란 1944년 미국 뉴햄프셔주 브레튼우즈에서 44개국이 모여 만든 국제 경제 질서로, 미국이 세계의 안전한 무역과 에너지 공급을 보장하는 대신 다른 나라들이 미국 중심의 질서에 참여하는 구조였다."

### 원칙 2: 중요한 부분은 예시를 적시하며 설명

- 핵심 주장, 중요한 개념, 인과관계 등에는 **반드시 구체적인 역사적·현실적 예시**를 들어 설명한다
- 예시는 추상적 서술이 아니라 구체적인 사건, 인물, 수치, 날짜를 포함한다
- 하나의 주장에 최소 1개 이상의 예시를 제시한다

> **예시**: "에너지 의존도가 국가 안보에 영향을 미친다"는 주장 → "2022년 러시아가 우크라이나를 침공했을 때, 독일은 천연가스의 약 40%를 러시아에서 수입하고 있었다. 이 때문에 독일은 초기에 강력한 대러 제재를 주저할 수밖에 없었고, 이는 에너지 의존이 곧 외교적 약점이 된다는 것을 보여준 대표적 사례다."

### 원칙 3: 적절한 곳에 표(Table)를 첨가

- 비교 분석, 항목 나열, 시계열 정리, 국가별·인물별 비교 등에는 **표를 적극적으로 활용**한다
- 표는 한눈에 핵심 정보를 파악할 수 있도록 간결하게 구성한다
- 복잡한 내용을 본문으로만 서술하지 말고, 표로 정리한 뒤 본문에서 보충 설명한다

> **예시**:
>
> | 국가 | 출산율 | 에너지 자급률 | 식량 자급률 | 종합 전망 |
> |------|--------|-------------|-----------|----------|
> | 미국 | 1.66명 | 100% 이상 | 130% | 최강 유지 |
> | 중국 | 1.0명 | 약 30% | 약 65% | 쇠퇴 위험 |
> | 한국 | 0.72명 | 약 5% | 약 45% | 구조적 취약 |

## 3. 에이전트(Subagent) 활용 가이드라인

작업의 품질과 효율성을 높이기 위해 **4종의 서브에이전트**를 활용하며, 각 에이전트는 명확한 역할 분담 하에 상호 연계(hook)하여 작동한다.

### 에이전트 구조 개요

```
┌─────────────────────────────────────────────────────┐
│                    작업 흐름                          │
│                                                      │
│  ① 조사 에이전트  ──→  ② 분석 에이전트               │
│        │                      │                      │
│        │                      ▼                      │
│        │              ③ 서술 에이전트                 │
│        │                      │                      │
│        │                      ▼                      │
│        └──────→  ④ 검증 에이전트  ──→ 최종 산출물    │
│                        │                             │
│                        ▼ (문제 발견 시)               │
│                  해당 에이전트로 피드백 반환            │
│                                                      │
└─────────────────────────────────────────────────────┘
```

### 에이전트별 역할 정의

#### ① 조사 에이전트 (Research Agent)

| 항목 | 내용 |
|------|------|
| **역할** | 주제에 관한 원천 자료를 수집하고 정리 |
| **수행 업무** | 웹 검색, 서적 정보 조사, 학자 견해 수집, 언론 보도 수집, 통계·수치 확보 |
| **산출물** | 조사 보고서 (원천 자료, 출처, 핵심 팩트 목록) |
| **연계** | 산출물을 ② 분석 에이전트에 전달 |

**조사 에이전트 작동 기준**:
- 최소 3개 이상의 출처를 확보한다
- 상반된 견해가 존재할 경우 양쪽 모두 수집한다
- 수치와 통계는 출처를 반드시 명시한다
- 조사 결과를 구조화된 형태(항목별 분류)로 정리한다

#### ② 분석 에이전트 (Analysis Agent)

| 항목 | 내용 |
|------|------|
| **역할** | 조사 에이전트가 수집한 자료를 분석하고 의미를 도출 |
| **수행 업무** | 인과관계 분석, 비교 분석, 패턴 식별, 시사점 도출, 예측 시나리오 구성 |
| **산출물** | 분석 프레임워크, 핵심 논점 정리, 비교표, 시나리오 |
| **연계** | 조사 에이전트의 산출물을 입력받고, 분석 결과를 ③ 서술 에이전트에 전달 |

**분석 에이전트 작동 기준**:
- 단순 나열이 아닌 **인과관계와 구조**를 파악한다
- 비교 가능한 항목은 반드시 표로 정리한다
- 핵심 주장에는 근거(evidence)를 명시한다
- 상반된 시각이 있을 경우 양측을 균형 있게 분석한다

#### ③ 서술 에이전트 (Writing Agent)

| 항목 | 내용 |
|------|------|
| **역할** | 분석 결과를 서술 원칙에 따라 최종 문서로 작성 |
| **수행 업무** | 문서 구조 설계, 본문 작성, 예시 삽입, 표 작성, 가독성 확보 |
| **산출물** | 완성된 마크다운 문서 |
| **연계** | 분석 에이전트의 산출물을 입력받고, 작성된 문서를 ④ 검증 에이전트에 전달 |

**서술 에이전트 작동 기준**:
- 서술 원칙 3가지(자세한 서술, 예시 적시, 표 활용)를 반드시 준수한다
- 문서 구조: 기본 정보 → 핵심 논지 → 장별/주제별 상세 분석 → 종합 평가
- 각 섹션에 최소 1개 이상의 구체적 예시를 포함한다
- 전문 용어는 초보자도 이해할 수 있도록 풀어서 설명한다

#### ④ 검증 에이전트 (Verification Agent)

| 항목 | 내용 |
|------|------|
| **역할** | 최종 산출물의 정확성, 완성도, 서술 원칙 준수 여부를 검증 |
| **수행 업무** | 팩트 체크, 서술 원칙 준수 확인, 논리적 일관성 검토, 누락 항목 점검 |
| **산출물** | 검증 보고서 (통과/수정 필요 판정, 수정 사항 목록) |
| **연계** | 서술 에이전트의 산출물을 입력받고, 문제 발견 시 해당 에이전트로 피드백 반환 |

**검증 에이전트 검증 항목**:

| 검증 항목 | 기준 |
|----------|------|
| **팩트 정확성** | 인물, 날짜, 수치, 사건명 등이 정확한가? |
| **서술 원칙 준수** | 초보자도 이해 가능한 수준인가? 예시가 충분한가? 표가 적절히 활용되었는가? |
| **논리적 일관성** | 주장과 근거가 논리적으로 연결되는가? 모순은 없는가? |
| **완성도** | 누락된 중요 항목은 없는가? 구조가 체계적인가? |
| **균형성** | 한쪽 시각에 치우치지 않았는가? 비판적 시각도 포함되어 있는가? |

### 에이전트 간 연계(Hook) 규칙

#### 순방향 연계 (Forward Hook)

```
조사 → 분석 → 서술 → 검증 → 최종 산출물
```

- 각 에이전트는 **이전 에이전트의 산출물을 입력**으로 받아 작업한다
- 이전 단계의 산출물이 불충분할 경우, 해당 에이전트에 추가 작업을 요청한다

#### 역방향 피드백 (Backward Hook)

```
검증 에이전트 → 문제 발견 시 → 해당 에이전트로 수정 요청 반환
```

- 검증 에이전트가 팩트 오류를 발견하면 → **조사 에이전트**에 재조사 요청
- 검증 에이전트가 분석 논리 오류를 발견하면 → **분석 에이전트**에 재분석 요청
- 검증 에이전트가 서술 원칙 미준수를 발견하면 → **서술 에이전트**에 재작성 요청

#### 병렬 연계 (Parallel Hook)

- 조사 에이전트가 여러 하위 주제를 동시에 조사할 경우, **복수의 조사 에이전트를 병렬 실행**한다
- 분석 에이전트가 여러 관점(찬성/반대, 국가별 비교 등)을 동시에 분석할 경우, **병렬 실행**한다
- 병렬 실행된 에이전트의 결과는 다음 단계 에이전트에서 **통합**한다

### 작업 유형별 에이전트 활용 예시

#### 서적 분석 시

| 단계 | 에이전트 | 수행 내용 |
|------|---------|----------|
| 1단계 | 조사 에이전트 | 서적 기본 정보, 저자 정보, 목차, 핵심 내용, 서평 조사 |
| 2단계 | 분석 에이전트 | 장별 핵심 논지 분석, 주장-근거 구조 파악, 비판적 시각 정리 |
| 3단계 | 서술 에이전트 | 서술 원칙에 따라 최종 분석 문서 작성 |
| 4단계 | 검증 에이전트 | 팩트 체크, 서술 원칙 준수 확인, 누락 항목 점검 |

#### 정치 상황 분석 시

| 단계 | 에이전트 | 수행 내용 |
|------|---------|----------|
| 1단계 | 조사 에이전트 (병렬) | A: 언론 보도 수집 / B: 학자 견해 수집 / C: 통계·데이터 수집 |
| 2단계 | 분석 에이전트 | 조사 결과 통합, 인과관계 분석, 시나리오 도출 |
| 3단계 | 서술 에이전트 | 분석 결과를 체계적 문서로 작성 |
| 4단계 | 검증 에이전트 | 최종 검증 및 피드백 |

## 4. 작업 절차 요약

```
1. 주제 접수
   ↓
2. 조사 에이전트 → 원천 자료 수집 (필요시 병렬 실행)
   ↓
3. 분석 에이전트 → 자료 분석 및 의미 도출
   ↓
4. 서술 에이전트 → 서술 원칙에 따른 문서 작성
   ↓
5. 검증 에이전트 → 정확성·완성도·원칙 준수 검증
   ↓ (문제 발견 시 해당 단계로 피드백 반환)
6. 최종 산출물 확정 → 커밋·푸시
```
